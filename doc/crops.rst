Adding new crops
----------------

New crops can be added in the GEOBIS crop model web service in the MySQL database.

Use the following steps to add a new crop:

1. Go to the *crop* table. Add a new crop_no (crop ID) and new crop_name (see red boxes figure below). **Click Apply**.

	.. image:: figures/Add_new_cropname_and_ID.png
	   :scale: 50 %
	   
	See `croptable`_ for more details.

2. Go to *crop_parameter_value* table and add crop properties of new crop at the bottom of the table (see red box figure below).

	The crop parameters consists of a number of lines that have to be provided. Each line includes crop_no, parameter_code, parameter_value and parameter_description.
	For each new row provide the crop_ID in column *crop_no* (see red labelled crop_no figure below).
	
	The first 3 rows must include the PHENO_TBASE, PHENO_TF and MAX_DURATION (see green box figure below):
	
	+--------------+----------------+--------------------------------------+----------------------------------+
	| crop_no      | parameter_code | parameter_value                      | parameter description            |
	+==============+================+======================================+==================================+
	| x            | PHENO_TBASE    | Tbase                                | Base temperature degrees Celsius |
	+--------------+----------------+--------------------------------------+----------------------------------+
	| x            | PHENO_TF       | e.g. [Tbase,0,Toptimum,1,Tmaximum,1] | Temperature function phen. dev.  |
	+--------------+----------------+--------------------------------------+----------------------------------+
	| x            | MAX_DURATION   | days                                 | Maximum length growing season    |
	+--------------+----------------+--------------------------------------+----------------------------------+
	
	The following rows include a number of different crop development stages (BBCH, from sowing to crop final stage) that need to be modelled with corresponding TSUM values (see blue box figure below):
	Which crop stages need to be included depends on the crop and management alerts. Typical BBCH values are:
	
		- BBCH_01 (seed sowing)
		- BBCH_10 (emergence)
		- BBCH_15 (5th leaf)
		- BBCH_61 (first flowering)
		- BBCH_81 (start of harvest period)
		- BBCH_89 (end of harvest period – last harvest, end of growing season)
    
	For each development stage (BBCH) a TSUM value (in degrees Celsius) has to be provided.	
	
	+--------------+----------------+--------------------------------------+----------------------------------+
	| crop_no      | parameter_code | parameter_value                      | parameter description            |
	+==============+================+======================================+==================================+
	| x            | BBCH_01        | TSUM = 0                             | Seed sowing                      |
	+--------------+----------------+--------------------------------------+----------------------------------+
	| x            | BBCH_10        | TSUM = ..                            | Emergence                        |
	+--------------+----------------+--------------------------------------+----------------------------------+
	| x            | BBCH_xx        | TSUM = ..                            | ....                             |
	+--------------+----------------+--------------------------------------+----------------------------------+
	| x            | BBCH_xx        | TSUM = ..                            | Harvest                          |
	+--------------+----------------+--------------------------------------+----------------------------------+
	
	|
	
	.. image:: figures/Add_new_crop_properties.png
	   :scale: 50 %
	   
	**Click Apply**.
	
	See `cropparametervalues`_ for more details.

3. Go to *management_alerts* table and add management alerts to the defined BBCH stages at the bottom of the table (see red box figure below).
	
	A number of management alerts can be defined, that are linked to specific crop development stages (BBCH). In turn the management alerts (msg_ID's) can be linked to SMS messages for farmers via the SMS panel (GEOBIS Aggregator platform).

4. Go to *weather_alerts* table and add weather alerts to the defined BBCH stages at the bottom of the table (see red box figure below).
	
	A number of weather alerts can be defined, that are linked to specific crop development stages (BBCH). In turn the weather alerts (msg_ID's) can be linked to SMS messages for farmers via the SMS panel (GEOBIS Aggregator platform).
	
	For now 5 different type of weather alerts are defined:

		- Heat stress alert (msg_id = 501)
		- Cold stress alert (msg_id = 502)
		- Humidity stress alert (msg_id = 503)
		- Rain stress alert (msg_id = 504)
		- Fog alert (msg_id = 505)
	
	Each weather alert includes crop_no, variety_no, message_no, parameters, weather_msg and a signal.
	
	+--------------+----------------+---------------+---------------------------------------------------------------------------------------------------+----------------+-------------------------+
	| crop_no      | variety_no     | message_no    |  parameters                                                                                       | weather_msg    | signal                  |
	+==============+================+===============+===================================================================================================+================+=========================+
	| x            | -1             | 501           | {'TMAX_CRIT_BBCH':["BBCH_xx",..],'TMAX_CRIT':[30,..],'TMAX_STRESS_DURATION':3}'                   | heat stress    | TMAX_STRESS             |
